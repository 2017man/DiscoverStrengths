## 二、对话背景与你的原始构想（摘要）

你希望做的**不仅仅是**一个测评网站《发现你的天赋优势》：**网站只是工具**，核心价值是**帮助别人找到优势天赋**，并以此为切入口；**最终目标是帮助更多的人解决困境与迷茫，与用户一起成长**，需要有跟别人不一样的亮点（有温度的成长陪伴，而非单纯卖报告）。

- **参考网站**（内容/结构可参考，风格不照搬）：talentsays.com、mbti9.hzshuf.cn、jyzx.jiuye.net、xmcs.cn、chinajob.mohrss.gov.cn/zy 等  
- **目标群体**：大学生、职场迷茫期、上升期想更精准、**以及想做副业但不知擅长什么的上班族**（找准兴趣点）  
- **产品形态**：多种测评工具（有分类）、题库 + 测试说明、温馨随机文字、报告可试看后付费；职业大典/职业数字展馆可做可不做；名人功能暂不做  
- **版本**：先做国内版（国内版双语，国外版暂不做）；国内版技术栈 uniapp + dcat-admin，国外版规划为 Next.js 全栈  
- **三阶段构想**：  
  - **v1.0**：MBTI、霍兰德、职业锚、DISC 四种测试的测试→结果→试看→付费→下载报告、SEO；付费定价如 8.88 元，先用易支付/微信支付  
  - **v2.0**：AI 组合结果（2/3/4 种组合定价）  
  - **v3.0**：职业库 + AI 职业匹配，帮助成长  
- **个人背景**：PHP 开发 6 年 → 央企 DICT 项目管理；打算将网站做成副业产品；爱人从事会计；是否注册公司、风险规避等需战略层面建议  

对话中经过多轮确认，对上述构想做了**收敛与细化**，形成下文共识。

## 三、产品定位与主张（共识）

| 项目 | 共识内容 |
|------|----------|
| **产品名称** | 发现你的天赋优势 |
| **主 Slogan** | 发现天赋，迈出第一步 |
| **副标** | 轮换使用两句：① 你的优势，值得被看见；② 天赋是起点，成长是旅程（用于报告页、落地页等增加温度） |
| **核心价值** | 帮助用户低成本迈出第一步——先发现自己的天赋与优势；后续通过社群与更多工具持续助力成长 |
| **产品调性** | 有温度、可信、专业、易上手 |

### 愿景、最终目标与差异化

- **不止是网站**：我们不仅仅是一个测评网站，**网站只是工具和入口**，不是终点。真正的目标是**帮助更多的人解决困境与迷茫，与用户一起成长**。
- **最终目标**：帮助更多人走出困境与迷茫，陪伴他们持续成长；形成「发现天赋 → 看见优势 → 持续成长」的完整陪伴。
- **差异化亮点**：与单纯卖测评报告的站点不同，我们的亮点在于**有温度的成长陪伴**——以测评为低成本第一步，以社群、公众号、更多工具与内容为延伸，真正关心「人」的困境与成长，而不只是交付一份报告。

## 五、版本与功能范围（共识）

### 5.1 版本路线

| 版本 | 目标 | 主要内容 |
|------|------|----------|
| **v1.0** | 跑通闭环、验证付费 | 仅 MBTI 完整闭环；首页 4 卡片（其余 3 灰）；SEO；定价后台可配置；易支付 |
| **v1.1** | 国内版双语 | 中文 + 英文切换（题库、选项、报告等） |
| **v2.0** | 多测评与组合分析 | 霍兰德、DISC、职业锚上线；AI 组合报告；2/3/4 种组合定价 |
| **v3.0** | 职业与成长（可选） | 职业大典/职业数字展馆可做可不做；AI 职业匹配；成长路径 |

### 5.2 v1.0 具体约定

- **只上线 MBTI**，先跑通「答题 → 报告（约 30% 预览）→ 付费解锁 → PDF 下载」整条闭环。  
- **首页展示 4 个测评卡片**：MBTI、霍兰德、DISC、职业锚；**仅 MBTI 可点击**，其余 3 个**灰色「即将上线」**，为后续预留入口。  
- **每种测评类型的价格在后台可配置**，前端与支付按配置读取，不写死。  
- **v1.0 仅中文**；国内版双语放在 **v1.1**。  
- **报告免费预览约 30%**；完整报告付费后可见、可下载 PDF。

### 5.3 职业大典

- **职业大典/职业数字展馆**：可做可不做，不做硬性承诺；若做则放在 v3.0 规划中。  
- 名人功能：暂不做。

## 十二、开发实现相关共识与决策（近期对话）

以下为产品/开发实现层面达成的共识与决策，供后续开发与对话延续使用。

### 12.1 数据库与数据导入

| 项目 | 共识内容 |
|------|----------|
| **表名统一前缀** | 所有业务表以 **strengths_** 为前缀（如 strengths_test_types、strengths_orders）。 |
| **关联方式** | 各业务表统一用 `test_type`（varchar，存 strengths_test_types.code，如 MBTI、HOLLAND），**不用 test_type_id**。 |
| **无 JSON 字段** | 表结构**彻底移除 JSON 字段**，便于 Navicat 从 CSV 直接导入、维护与查询。 |
| **结果表合并** | 十六种性格类型由原 test_result_codes + test_result_details **合并为一张表 strengths_test_answer**，存完整报告内容；用户单次测试记录存 **strengths_test_results_records**。 |
| **维度与面** | 03-答案-MBTI的四个维度.csv → **strengths_test_dimensions**；03-答案-四个维度八个面.csv → **strengths_test_dimension_sides**，各为独立表。 |
| **评分规则** | 04-评分规则.csv **不建表**，以「评分规则说明」形式写在《数据库设计》中，开发时查阅。 |
| **导入方式** | 题目与人格数据以 **Navicat 从 CSV 导入** 为主；CSV 与表列对应、推荐导入顺序见《CSV与表结构说明》。 |

### 12.2 前端开发节奏与风格

| 项目 | 共识内容 |
|------|----------|
| **先 UI 后接口** | 前端页面**先不绑定接口**，使用静态/模拟数据完成 UI；等产品负责人**确认前端页面 UI 后再进行接口绑定**。 |
| **页面风格** | 所有页面贯穿 **大气、简洁、有质感、用心**；留白充足、层级清晰、按钮与反馈明确、文案简洁有温度。 |
| **旧页面定位** | `pages/h5/evaluate/` 下原有页面（如 index、PageHeader）**仅作接口调用方式与结构参考**，**视觉与版式按新风格重新设计**，不沿用旧项目 UI。 |

### 12.3 前端 Web 与 H5 兼容

| 项目 | 共识内容 |
|------|----------|
| **双端兼容** | 站点需**同时兼容 Web 端（桌面浏览器）与 H5 端（手机浏览器）**，同一套代码两端可用。 |
| **Web 大屏** | 视口宽度 ≥ 751px 时，页面根容器使用类名 **page-wrap-web**，全局限制 **max-width: 750px**、居中、轻微阴影，呈「居中卡片」效果。 |
| **H5 小屏** | 宽度 &lt; 751px 不限制最大宽度，rpx 自适应，保持移动端全屏体验。 |
| **交互** | H5 依赖 `:active`、触控区域足够大；Web 为主要按钮/卡片增加 `:hover`，保证鼠标反馈。 |
| **安全区域** | PageHeader 预留 **safe-area-inset-top**；答题页底部栏预留 **safe-area-inset-bottom**，适配刘海屏与 Home 条。 |

## 十三、首页「关注我们」引流与文案共识（近期对话）

以下为首页公众号/社群引流、文案与样式达成的共识与决策。

### 13.1 为什么要关注公众号 / 为什么要加入社群

| 入口 | 定位与价值 |
|------|------------|
| **公众号** | 定位**个人成长 + 副业**；持续输出成长干货、职业规划、副业思路、性格解读等；用户得到的是一路陪伴的内容入口，而非一次性报告。 |
| **社群** | **持续服务、长期陪跑、群体优惠**；进来之后有答疑、专属活动、社群优惠价、同路人陪伴，从单次付费延伸到长期关系。 |

### 13.2 暖心话术与口语化原则

- **总领句**：想和你一起，走得更远。  
- **共鸣句（暖心段落）**：不知道副业怎么开始？不知道怎么从 0 到 1？不知道自己该从哪里下手去改变？不知道个人成长怎么破局、怎么踏出第一步？甚至担心，踏出第一步之后也没法持续……**没关系，我们长期陪伴。**  
- **副句**：在这里，你不必一个人摸索。  
- **公众号卡片**：口语化、有温度。示例：「我的公众号就做两件事：**个人成长**和**副业**。会持续写怎么认识自己、怎么规划、怎么从 0 开始一点一点变好。关注一下，成长路上多一个伴。」  
- **社群卡片**：口语化、有诚意。示例：「进来之后不只是一次性服务。会有**持续的各种支持**、**长期陪跑**，还有**群体专属优惠**。有问题可以问，有同路人可以聊，我们一起往前走。」  
- **原则**：不干巴巴，**口语化、有温度、有诚意**；重点内容用主色强调（如个人成长、副业、长期陪跑等）。

### 13.3 文案结构：三段式

- **公众号**：① 定位（两件事：个人成长和副业）；② 内容（怎么认识自己、规划、从 0 变好）；③ 行动/情感（关注一下，成长路上多一个伴）。  
- **社群**：① 定位（不只是一次性服务）；② 价值（持续支持、长期陪跑、群体专属优惠）；③ 行动/情感（有问题可问、同路人可聊，一起往前走）。  
- 每段独立成行（如 `.desc-line`），便于阅读与后续微调。

### 13.4 样式与布局共识

| 项目 | 共识内容 |
|------|----------|
| **区块位置** | 首页底部「关注我们」在测评卡片之后、footer 之前；不改变 `follow-section` 的 padding / max-width。 |
| **暖心段落** | 问句 + 收束句「没关系，我们长期陪伴」可放在轻主色背景块内，收束句略大、主色加粗，更打眼。 |
| **卡片** | 两卡并排（桌面端）；可带主色顶条、轻阴影；**单卡桌面端 max-width: 360rpx**，避免两卡过宽。 |
| **两卡间距** | 仅通过 `.follow-cards` 的 **gap** 或卡片 **margin** 控制，**不改变 follow-section**；桌面端约 **8rpx** 级，移动端竖排约 8rpx，两卡更紧凑。 |
| **二维码** | 二维码区域 **与卡片同宽**（`width: 100%`、`align-self: stretch`），铺满卡片底部，圆角与卡片底部一致；两卡等宽时两处二维码宽度一致。 |

## 十七、MBTI 答题页共识（近期对话）

以下为 MBTI 答题页（题目页）进度展示、答题须知形式达成的共识与决策。

### 17.1 答题须知

| 项目 | 共识内容 |
|------|----------|
| **文案内容** | 两段完整说明：① MBTI 无好坏优劣、与能力/道德无关，按**真实自己**而非期待中的自己作答；② 无需过分思考，遇纠结正常，选**放松时最可能的倾向**。 |
| **展示位置** | 答题页（questions.vue）内、进度条下方、题目上方；intro 页可保留或移除，以答题页为准。 |
| **交互形式** | **可折叠**：默认收起，只显示「答题须知」+ 简短一句「按真实自己作答，遇纠结选放松时倾向」+ 箭头 ▶/▼；点击展开显示两段全文。 |
| **样式** | 浅主色背景、左侧主色竖条（6rpx）、圆角；展开后正文与标题区有分隔线。 |

### 17.2 进度展示

| 项目 | 共识内容 |
|------|----------|
| **设计原则** | 进度条**一条线**表示完成比例，**末端一个圆点**表示「当前进度位置」；不采用多圆点/多段式，避免信息重复、视觉杂乱。 |
| **视觉** | 一条横向轨道（灰色）+ 已完成部分（主色填充）+ **仅在进度末端的一个圆点**（主色），圆点随进度移动。 |
| **文案** | 下方显示「已答 x / 共 n 题」+ **百分比**，如「已答 12 / 共 93 题（13%）」，题量与百分比由 `questions.length` 与已答数计算，适配 5 题或 93 题。 |
| **与头部关系** | 进度区与页面头部（PageHeader）之间留有明显间距（如 margin-top 48rpx），不贴在一起。 |

### 17.3 与开发/文档的对应

- 题目数据为 5 题 mock 或 93 题时，进度逻辑均按「已答数/总题数」计算，无需为 93 题单独改进度条结构。  
- 答题须知折叠状态用 `noticeExpanded` 控制，默认 `false`。
