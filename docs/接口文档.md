# 《发现你的天赋优势》接口文档

## 一、约定说明

- **Base URL**：开发环境 `http://collect.com/api/v1`（与前端 request.js 一致）；生产环境由部署配置。
- **请求头**：`Content-Type` 见各接口说明；`Authorization` 可选（H5 匿名答题可不传，支付/订单可传 token 或 openid 等）。
- **统一响应**：
  - 成功：`{ "code": 200, "msg": "success", "data": { ... } }`
  - 业务错误：`{ "code": 10000+, "msg": "错误说明" }`
- **文档中路径**：均相对于 Base URL，如 `test-types/list` 即 `GET /api/v1/test-types/list`。

## 二、测试类型相关

### 2.1 测试类型列表（首页 4 卡片）

- **路径**：`GET /test-types/list`
- **说明**：返回所有测试类型（含 MBTI、霍兰德、DISC、职业锚），用于首页展示；仅 MBTI 可点时由前端根据 `code` 或 `status`/`is_available` 控制灰色与「即将上线」。
- **请求参数**：无（或可选 `status=1` 只返回启用）。
- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": 1,
      "code": "MBTI",
      "name": "MBTI 职业性格测试",
      "description": "简要说明",
      "total_questions": 93,
      "sort": 1,
      "status": 1,
      "price": "8.88",
      "is_available": true
    },
    {
      "id": 2,
      "code": "HOLLAND",
      "name": "霍兰德职业兴趣测试",
      "total_questions": 0,
      "sort": 2,
      "status": 1,
      "price": "8.88",
      "is_available": false
    }
  ]
}
```

- **字段说明**：`price` 为后台可配置单价（元）；`is_available` 为是否已上线可测（v1.0 仅 MBTI 为 true）。

## 三、MBTI 题目与说明

### 3.1 MBTI 测试说明（说明页）

- **路径**：`GET /mbti/intro` 或 `GET /test-types/detail?code=MBTI`
- **说明**：获取 MBTI 的说明文案、题数、建议时长等，用于说明页展示。
- **请求参数**：无；或 `code=MBTI`。
- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": 1,
    "code": "MBTI",
    "name": "MBTI 职业性格测试",
    "description": "测试说明正文...",
    "total_questions": 93,
    "price": "8.88",
    "estimate_minutes": 15
  }
}
```

### 3.2 MBTI 题目列表（答题页）

- **路径**：`GET /mbti/questions`
- **说明**：获取 MBTI 全部题目，按题号排序；选项由后端从 **strengths_test_question_options** 按题组装为数组返回；同时返回题目分组 **sections**（来自 **strengths_test_questions_section**），前端可按 `section_code` 分组展示、用 `section_title` 作当前块引导语。表结构见《数据库设计》。
- **请求方式**：GET
- **请求参数**：无。
- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": 1,
      "question_number": 1,
      "question_text": "在聚会中，你通常",
      "section_code": "一",
      "dimension": "E-I",
      "options": [
        { "key": "A", "text": "与很多人交谈，包括陌生人", "side": "E" },
        { "key": "B", "text": "只与少数熟人交谈", "side": "I" }
      ]
    },
    {
      "id": 2,
      "question_number": 2,
      "question_text": "你更倾向于",
      "section_code": "一",
      "dimension": "S-N",
      "options": [
        { "key": "A", "text": "按已有方式做事", "side": "S" },
        { "key": "B", "text": "尝试新方法", "side": "N" }
      ]
    }
  ],
  "sections": [
    {
      "section_code": "一",
      "section_title": "哪一个答案最能贴切的描绘你一般的感受或行为?",
      "sort": 0
    },
    {
      "section_code": "二",
      "section_title": "在下列每一对词语中，哪一个词语更吸引你？",
      "sort": 1
    },
    { "section_code": "三", "section_title": "题目标题/引导语…", "sort": 2 },
    { "section_code": "四", "section_title": "…", "sort": 3 }
  ]
}
```

- **响应字段说明**

| 层级 | 字段     | 类型   | 说明                                                                  |
| ---- | -------- | ------ | --------------------------------------------------------------------- |
| 根级 | code     | number | 200 表示成功                                                          |
| 根级 | msg      | string | 提示信息，成功时为 "success"                                          |
| 根级 | data     | array  | 题目列表，按 question_number 升序                                     |
| 根级 | sections | array  | 题目分组列表，按 sort 升序；用于按分组展示、取 section_title 作引导语 |

- **data[] 中每题对象**

| 字段            | 类型   | 说明                                                                                                                           |
| --------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------ |
| id              | number | 题目主键（strengths_test_questions.id）                                                                                        |
| question_number | number | 题号，提交答案时与 option_key 一起传                                                                                           |
| question_text   | string | 题目正文                                                                                                                       |
| section_code    | string | 所属分组代码（如一、二、三、四），与 sections[].section_code 对应                                                              |
| dimension       | string | 所属维度，与数据库 dimension_code 一致：**E-I**（外向/内向）、**S-N**（感觉/直觉）、**T-F**（思考/情感）、**J-P**（判断/感知） |
| options         | array  | 选项列表，见下表                                                                                                               |

- **data[].options[] 选项对象**

| 字段 | 类型   | 说明                                                                                    |
| ---- | ------ | --------------------------------------------------------------------------------------- |
| key  | string | 选项键（如 A、B），提交答案时传 **option_key** 与此一致                                 |
| text | string | 选项文案                                                                                |
| side | string | 该选项对应的维度面（E/I/S/N/T/F/J/P），来自 strengths_test_question_options.dimension_side |

- **sections[] 分组对象**

| 字段          | 类型   | 说明                                      |
| ------------- | ------ | ----------------------------------------- |
| section_code  | string | 分组代码（一、二、三、四）                |
| section_title | string | 分组标题/引导语，答题页可用作当前块的说明 |
| sort          | number | 排序，数值越小越靠前                      |

- **使用说明**：
  - 计分在后端完成，前端只需在提交时传 **question_number** + **option_key**（与 options[].key 一致），见 [4.1 提交答案并获取结果](#41-提交答案并获取结果)。
  - 答题页可按 **section_code** 将题目分组展示；用 **sections** 中对应项的 **section_title** 作为当前块的引导语，tab 可用 **section_code** 或 section_title 简短文案展示。
  - **dimension** 仅作展示参考（如标注维度 E-I/S-N/T-F/J-P），不参与前端计分。

## 四、MBTI 答题与结果

### 4.1 提交答案并获取结果

- **路径**：`POST /mbti/submit`
- **Content-Type**：`application/json` 或 `application/x-www-form-urlencoded`
- **说明**：提交用户答案，后端按《数据库设计》中「MBTI 测试逻辑」计分（用 strengths_test_question_options.dimension_side 汇总），得到 16 型之一，写入 **strengths_test_results_records**（含 e_score、i_score、s_score、n_score、t_score、f_score、j_score、p_score 八个面得分）；报告内容从 **strengths_test_answer** 按 test_type+result_code 查询；返回结果码与预览内容。
- **请求体示例**：

```json
{
  "answers": [
    { "question_number": 1, "option_key": "A" },
    { "question_number": 2, "option_key": "B" }
  ]
}
```

或按题号顺序：`{ "answers": [ { "question_number": 1, "option_key": "A" }, ... ] }`。后端用 question_number + option_key 在 **strengths_test_question_options** 中取 dimension_side 计分。

- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "result_id": 1001,
    "result_code": "INTJ",
    "result_name": "建筑师",
    "preview_content": "免费预览的 30% 内容（核心特征、一句话总结等）",
    "is_paid": false
  }
}
```

- **说明**：同一用户（如 openid）+ 同一设备/会话在未付费前可复用 `result_id`；若该 result 已付费，再次请求可返回 `is_paid: true` 及完整内容标识。

### 4.2 报告详情（预览 / 完整）

- **路径**：`GET /mbti/report?result_id=1001`
- **说明**：根据 `result_id` 返回报告内容；未付费时仅返回 `preview_content` 等预览字段；已付费时返回完整报告所需的结构化字段（见下表）。前端完整报告页按「第 1 ～ 6 部分 + 性格概述 + 沟通与成长建议 + 温馨寄语 + 一起成长」布局渲染。
- **请求参数**：
  - `result_id`（必填）
  - `full`（可选，传 `1`）：**测试环境**，前端带 `full=1` 时请求完整报告；后端可据此在未付费时也返回完整结构化字段，便于联调与预览。

**响应字段说明**（已付费完整报告需支持以下结构化字段，未付费可只返回基础字段）：

| 字段                                                 | 类型            | 说明                                                                                                                                                                                                                                                                                       |

| result_id                                            | number          | 结果 ID                                                                                                                                                                                                                                                                                    |
| result_code                                          | string          | 类型代码，如 INTJ、ISTJ                                                                                                                                                                                                                                                                    |
| result_name                                          | string          | 类型名称，如「建筑师」「守护者」                                                                                                                                                                                                                                                           |
| summary                                              | string          | 总括/一句话描述（来自 strengths_test_answer.summary，兼容 result_desc）                                                                                                                                                                                                                    |

| price                                                | string          | 单价（未付费时展示）                                                                                                                                                                                                                                                                       |
| preview_content                                      | string          | 预览内容（约 30%，未付费时用）                                                                                                                                                                                                                                                             |
| **完整报告结构化字段**（is_paid 为 true 时建议返回） |                 |                                                                                                                                                                                                                                                                                            |
| dimension_scores                                     | array           | 四维度得分，用于「第 1 部分·性格倾向明显程度示意图」。由 strengths_test_results_records 的 e_score ～ p_score 计算百分比。每项：`{ dimension_code, leftCode, rightCode, leftScore, rightScore }`，**dimension_code** 与数据库一致：`"E-I"`/`"S-N"`/`"T-F"`/`"J-P"`，顺序固定，每维度唯一。 |
| dimension_sides                                      | array           | 八个面说明（来自 strengths_test_dimension_sides）。每项：`{ side_code, side_name }`，8 条唯一（E/I/S/N/T/F/J/P）。用于第 1 部分图表标签、第 2 部分名称兜底。                                                                                                                               |
| dimensions                                           | array           | 四个维度元数据（来自 strengths_test_dimensions）。每项：`{ dimension_code, dimension_scope }`，4 条唯一（E-I/S-N/T-F/J-P）。**仅当未返回 dimensions_detail 时**，用于第 2 部分维度标题。                                                                                                   |

| typical_characters / typical_figures                 | string \| array | 典型人物（strengths_test_answer.typical_figures）。可为字符串或字符串数组；后端可返回 typical_figures，前端兼容两者                                                                                                                                                                        |

| weaknesses                                           | array           | 劣势列表，用于「第 5 部分」。如 `["不愿接受新观念", "抗拒变化", …]`                                                                                                                                                                                                                        |

**strengths / weaknesses / traits / careers 数据格式说明**：

| 字段 | 数据库存储格式 | 接口返回格式 | 前端展示建议 |

## 五、订单与支付

### 5.1 创建订单（获取支付参数）

- **路径**：`POST /order/create`
- **Content-Type**：`application/json`
- **说明**：根据 **result_id** 创建订单；后端从 strengths_test_results_records 取 test_type、校验未付费与归属，金额从 strengths_test_types（code=test_type）取 price，返回易支付跳转 URL 或支付参数。
- **请求体示例**：

```json
{
  "result_id": 1001
}
```

- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "order_id": "ORD202401010001",
    "amount": "8.88",
    "pay_url": "https://xxx/pay?out_trade_no=...",
    "out_trade_no": "xxx",
    "epay_order_id": "201901102220147500"
  }
}
```

- **字段说明**：`epay_order_id` 为易支付云端订单号（可选返回），用于轮询支付状态接口；前端若需轮询，请保存 `out_trade_no`。
- **说明**：前端跳转 `pay_url` 完成支付；支付成功后易支付回调后端，后端更新订单状态并标记该 result 已付费。
- **测试阶段免支付**：当后端开启 `PAYMENT_SKIP_FOR_TEST` 时，响应中会多出 `data.skip_payment: true`，且 `pay_url` 为前端报告页地址（带 result_id）；订单与 result 已直接标记为已支付，前端可据此直接跳转报告页或提示「测试模式已免支付」。详见 [易支付配置说明](./易支付配置说明.md)。

### 5.2 支付回调（后端对易支付）

- **说明**：由后端接收易支付异步回调，校验签名、更新订单状态、更新测试结果付费状态；不在此文档规定前端调用。

### 5.3 订单状态 / 是否已付费

- **路径**：`GET /order/status?result_id=1001` 或 `GET /mbti/report?result_id=1001`（报告接口内已含 `is_paid`）
- **说明**：若报告接口已返回 `is_paid`，可不再单独查订单；否则提供本接口用于轮询或进入报告页前校验。
- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "result_id": 1001,
    "is_paid": true,
    "order_id": "ORD202401010001"
  }
}
```

### 5.4 轮询支付状态（易支付 checkOrder）

- **路径**：`GET /order/check-payment?out_trade_no=xxx`
- **说明**：根据商户订单号查询易支付侧是否已支付；后端会调用易支付 checkOrder，若已支付则同步更新本地订单与测试记录。**适用场景**：支付页展示二维码（isHtml=0）时，前端保存创建订单返回的 `out_trade_no`，定时轮询本接口，当 `paid === true` 时提示支付成功并跳转报告页；或用户从易支付同步跳转回报告页后，若异步回调尚未到达，可用本接口拉齐本地状态。
- **请求参数**：`out_trade_no`（必填，即创建订单返回的 `out_trade_no`）。
- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "out_trade_no": "DS202402101200001234",
    "paid": true,
    "state": 1,
    "message": "已支付"
  }
}
```

- **data 字段说明**：`paid` 是否已支付；`state` 易支付状态（1=已支付，-1=失败，-2=未支付，-3=已过期）；`message` 说明文案。

## 七、站点配置（首页）

### 7.1 站点配置

- **路径**：`GET /config/site`
- **说明**：返回首页展示所需的测试总人次、日期、公众号与社群二维码 URL。
- **请求参数**：无。
- **响应示例**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "stats_count": "12345",
    "stats_date": "2014年5月19日 ~ 至今",
    "qrcode_wechat": "https://xxx/wechat-qr.png",
```
